UNCERTAINTY MODE — SPARSE-DATA DOMINATED

Research behavior:
- maximize value of each datapoint
- use scenario bands instead of point estimates
- rank unknowns by decision relevance

Risk:
- paralysis from missing data

## Stage Relevance

Routing — Primary
- Classifies the problem as one where key data is delayed, missing, or unreliable.

Boundary — Light
- Can widen scope slightly to include substitute signals and coarse proxies.

Structure — Primary
- Forces the stage to separate what is unknowable from what is merely delayed.

Scenarios — Primary
- Favors wider bands, coarse branches, and reduced point-estimate confidence.

Question Generation — Primary
- Favors minimum-evidence, proxy, and decision-under-missing-data questions.

Evidence Planning — Primary
- Requires explicit hierarchy for scarce datapoints and careful proxy substitution.

Decision Logic — Primary
- Pushes action logic toward minimum robust action and avoidance of false certainty.

Signal Translation — Primary
- Signals should be high-value, sparse, and event-driven rather than constantly refreshed.

Monitoring — Primary
- Monitoring should be event-driven with clear update thresholds.

Render — Modulating
- Should make data gaps and their consequences explicit.
