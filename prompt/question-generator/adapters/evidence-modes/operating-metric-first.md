EVIDENCE MODE — OPERATING-METRIC-FIRST

Prioritize:
- churn
- activation
- conversion
- usage depth
- throughput
- margins
- capacity
- cohort behavior
- time-to-value

Strengths:
- close to mechanism

Weaknesses:
- incomplete data
- KPI selection bias

## Stage Relevance

Routing — Primary
- Selects an operating-metric-first proof style close to mechanism.

Boundary — Light
- Slightly centers scope on the parts of the system that emit usable operating data.

Structure — Modulating
- Encourages mechanism claims that map directly to behavior, throughput, retention, and unit economics.

Scenarios — Modulating
- Prefers scenarios that can be distinguished by operational metrics moving first.

Question Generation — Primary
- Favors questions about which metric moves first if the thesis is right or wrong.

Evidence Planning — Primary
- Sets KPI hierarchy, proxy tolerance, and conflict handling around operating metrics.

Decision Logic — Modulating
- Can support action when operating evidence is stronger than narrative but still incomplete.

Signal Translation — Primary
- Favors churn, cohorts, activation, throughput, capacity, and margin signals.

Monitoring — Primary
- Monitoring should track high-frequency operating metrics and deterioration or acceleration patterns.

Render — Light
- Mainly affects whether the output explicitly shows KPI logic.
