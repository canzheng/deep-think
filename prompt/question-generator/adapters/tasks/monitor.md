TASK ADAPTER — MONITOR

Prioritize:
- leading indicators
- signal density
- thresholds
- update cadence
- noise filtering

## Stage Relevance

Routing — Primary
- Sets the overall job to ongoing watch rather than one-shot explanation or decision.

Boundary — Primary
- Pulls scope toward live variables, update cadence, and the systems that emit diagnostic signals.

Structure — Primary
- Prioritizes signal-generating structure and threshold behavior.

Scenarios — Primary
- Requires competing live paths that monitoring can distinguish in real time.

Question Generation — Primary
- Favors signal-selection questions, threshold questions, and update-rule questions.

Evidence Planning — Modulating
- Prefers early, high-signal, repeatable evidence over exhaustive evidence.

Decision Logic — Light
- Matters when monitoring is tied to escalation or action triggers.

Signal Translation — Primary
- This is a core stage for the adapter and should be first-class.

Monitoring — Primary
- This is the core operational stage: cadence, escalation, action triggers, and noise filtering.

Render — Primary
- Pushes the final artifact toward dashboard-like outputs and concise update logic.
