TASK ADAPTER — DECIDE

Prioritize:
- action relevance
- payoff asymmetry
- timing
- triggers
- failure modes
- minimum sufficient evidence

## Stage Relevance

Routing — Primary
- Sets the overall job to determining what action should be taken.

Boundary — Primary
- Pulls scope toward payoff-relevant factors, timing constraints, and practical decision context.

Structure — Primary
- Prioritizes decision-relevant structure, not just causal completeness.

Scenarios — Primary
- Requires scenarios that matter for action, downside, timing, and failure modes.

Question Generation — Primary
- Favors questions whose answers would change action, timing, sizing, staging, or hedging.

Evidence Planning — Primary
- Pushes the proof standard toward minimum sufficient evidence to act.

Decision Logic — Primary
- This is a core stage for the adapter and should make action thresholds explicit.

Signal Translation — Modulating
- Signals should indicate when to act, wait, add, trim, hedge, or stop.

Monitoring — Modulating
- Monitoring should track action triggers, not only belief updates.

Render — Primary
- Pushes the final artifact toward recommendation, evidence threshold, and triggers.
