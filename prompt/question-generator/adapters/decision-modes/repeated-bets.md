DECISION MODE — REPEATED BETS

Use when:
- many similar decisions will occur
- edge and calibration matter more than perfection on one case
- losses are survivable if sized correctly

Research behavior:
- lower the evidence bar per bet, but raise the bar for process discipline
- focus on base rates, hit rate, payoff ratio, and edge decay
- separate process quality from short-term outcome

Key questions:
- What is the repeatable edge?
- What is the base rate?
- What is the payoff distribution?
- How quickly does the edge decay?
- What sizing rule is appropriate?

Action logic:
- process > one outcome
- calibrate continuously
- size so survival is preserved

Monitoring style:
- frequent, process-based, outcome-aware

Failure mode:
- overconfidence from small samples

## Stage Relevance

Routing — Primary
- Classifies the action problem as repeatable and calibration-driven rather than one-off.

Boundary — Light
- Pulls in base rates, repeatability, and sizing discipline.

Structure — Modulating
- Adds edge durability, payoff ratio, and process quality to the structural view.

Scenarios — Primary
- Favors branches tied to payoff distribution and edge decay, not only single-case outcomes.

Question Generation — Primary
- Favors questions about repeatable edge, base rate, and sizing rules.

Evidence Planning — Modulating
- Lowers per-bet evidence demands while strengthening process-discipline evidence.

Decision Logic — Primary
- Core stage: should define sizing, calibration, and process rules.

Signal Translation — Primary
- Signals should indicate edge persistence, decay, and process drift.

Monitoring — Primary
- Monitoring should be frequent and process-based, not merely outcome-chasing.

Render — Modulating
- Outputs should make repeatability and process discipline explicit.
